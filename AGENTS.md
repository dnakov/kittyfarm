## Build & Run

This is an Xcode project generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen) from `project.yml`. Regenerate the Xcode project after editing `project.yml`:

```sh
xcodegen generate
```

Build from the command line:

```sh
xcodebuild -project KittyFarm.xcodeproj -scheme KittyFarm -configuration Debug build
```

The app targets **macOS 26.0** with **Swift 6.0** (strict concurrency). Code signing is disabled (`CODE_SIGNING_ALLOWED: NO`).

### KittyProbe (embedded iOS test helper)

A separate iOS XCUITest project lives at `KittyProbe/` (own `project.yml`, own `KittyProbe.xcodeproj`). It's a tiny host app plus a UI test target that serves the live accessibility tree of any foreground iOS app over a local HTTP endpoint (`/ping`, `/tree?bundleId=...`) ‚Äî used by the testing framework.

KittyFarm's `Build and embed KittyProbe` postCompile script (in `project.yml`) runs `xcodebuild build-for-testing` against `KittyProbe.xcodeproj` into `/tmp/KittyProbeBuilds/` and copies the products (`KittyProbeApp.app`, `KittyProbeTests-Runner.app`, `.xctestrun`) into `KittyFarm.app/Contents/Resources/KittyProbe/Debug-iphonesimulator/`. The `.xctestrun` uses `__TESTROOT__` placeholders so it's portable ‚Äî the embedded copy works with `xcodebuild test-without-building` without path rewriting. The script uses `env -i` to isolate the child `xcodebuild` from the parent Xcode build environment (otherwise leftover bundle references cause fake "missing file" errors).

Regenerate the probe project after editing its sources: `cd KittyProbe && xcodegen generate`.

## Proto Codegen (Android gRPC)

The Android emulator connection uses gRPC stubs generated from `Protos/emulator_controller_kittyfarm.proto` (a subset of the upstream `emulator_controller.proto`). Regenerate with the command in `KittyFarm/Generated/GeneratedNotice.swift` ‚Äî adjust the `--plugin` paths to your local `protoc-gen-swift` and `protoc-gen-grpc-swift-2` binaries. Output lands in `KittyFarm/Generated/`.

## Architecture

KittyFarm is a macOS SwiftUI app that displays and controls multiple iOS Simulators and Android Emulators side-by-side. A "leader" device can replicate touch input to all other devices simultaneously.

### Key abstractions

- **`DeviceDescriptor`** ‚Äî enum identifying a device (iOS simulator by UDID, or Android emulator by AVD name + gRPC port).
- **`DeviceConnection`** protocol + **`AnyDeviceConnectionBox`** type-eraser ‚Äî uniform interface for connecting, disconnecting, sending input, and device actions (home, rotate, openApp).
  - **`IOSSimulatorConnection`** ‚Äî uses `PrivateSimulatorDisplayBridge` (Obj-C, bridges private SimulatorKit frameworks) to attach to a simulator's display and inject touches via normalized coordinates. `openApp` parses `xcrun simctl listapps` (OpenStep plist) and launches via `simctl launch`.
  - **`AndroidEmulatorConnection`** ‚Äî uses **`GRPCFrameService`** (streams screenshots) and **`GRPCTouchInjector`** (sends touch events) over the emulator's local gRPC endpoint. Home/rotate/openApp go through `adb` (no dedicated RPCs in our proto subset for these ‚Äî upstream has `setPhysicalModel` for rotation but we haven't regenerated stubs for it).
- **`ADBUtils`** ‚Äî centralized `adb` and `emulator` binary path resolution + `adb devices` / `adb emu avd name`-based serial mapping from an AVD name to an `emulator-XXXX` serial. Used by `BuildPlayRunner`, `RuntimeLogStreamManager`, `EmulatorManager`, and the testing providers instead of duplicating the SDK-path logic.
- **`KittyFarmStore`** (`@Observable`, `@MainActor`) ‚Äî central app state: available/active devices, leader selection, sync toggle, bottom panel state, test script state. Owns all connections and delegates touch replication to `InputCoordinator`.
- **`InputCoordinator`** ‚Äî fans out a `NormalizedTouch` to multiple connections concurrently via a task group.

### iOS Simulator display path

The Obj-C bridge (`DFPrivateSimulatorDisplayBridge`) uses private SimulatorKit APIs to get a live `NSView` of the simulator display. This avoids ScreenCaptureKit and works without screen recording permissions. The bridging header is at `KittyFarm/Support/KittyFarm-Bridging-Header.h`.

### Android Emulator display path

`GRPCFrameService` opens a `streamScreenshot` gRPC call requesting RGBA8888 frames. Incoming frames are converted from RGB/RGBA to BGRA (for Metal) and pushed to `DeviceState`. `GRPCTouchInjector` opens a new gRPC connection per touch event, mapping normalized coordinates to pixel coordinates based on the current frame size.

### Rendering

`MetalFrameView` renders device frames (either `CVPixelBuffer` from iOS or raw RGBA `Data` from Android) using Metal. iOS simulator displays can alternatively use the private `NSView` directly via `SimulatorDisplayView`.

### Testing framework (`KittyFarm/Testing/`)

A natural-language test DSL that runs against connected devices. Scripts look like:

```
open "Settings"
tap "General"
wait for "About"
type "Hello" in "Name"
swipe up on "Chat list"
assert visible "Delivered"
```

**Execution flow** (`TestRunner`): for each action, fetch a fresh accessibility tree via the platform's `AccessibilityTreeProvider`, match the quoted string to an element via `ElementResolver`, convert its frame center to normalized 0‚Äì1 coordinates, then dispatch through existing `DeviceConnection.sendTouch` / `sendKey` / `setPasteboardText` / `pressHomeButton` / `openApp`. Multi-device runs execute in parallel via `withTaskGroup`; a `TestResultCollector` actor safely funnels per-step results back to the `@MainActor` store.

**Parsing** (`TestScriptParser`): line-oriented regex-free parser with smart-quote normalization (macOS `TextEditor` auto-substitutes `"` with `"`/`"`, which we map back to straight quotes before parsing). `#` lines are comments.

**Element resolution** (`ElementResolver`): flatten the tree, filter out zero-sized nodes, rank candidates by match quality (exact label > case-insensitive exact > case-insensitive contains; ties broken by interactive elements first, then shorter labels).

**iOS tree discovery** (`IOSAccessibilityProvider`): starts KittyProbe from the embedded bundle via `xcodebuild test-without-building` with the embedded `.xctestrun`, parses stdout for `[KittyProbe] port=‚Ä¶`, then GETs `/tree?bundleId=X`. `test-without-building` handles app installation via the `.xctestrun` ‚Äî do *not* call `simctl install` separately, it conflicts. The probe's main thread must stay on a run loop (`RunLoop.current.run()`), not a `DispatchSemaphore.wait()`, because the `/tree` handler dispatches to the main thread for `XCUIApplication.snapshot()` ‚Äî the semaphore would deadlock.

**Android tree discovery** (`LazyAndroidAccessibilityProvider`): `adb shell "uiautomator dump /sdcard/kittyfarm_ui.xml && cat /sdcard/kittyfarm_ui.xml"` (the `/dev/tty` redirect in the stock uiautomator example doesn't work over adb). XML parsed with Foundation's `XMLParser`; `<hierarchy>`'s direct children are promoted to root elements so the tree isn't empty. ADB serial resolved lazily via `ADBUtils.resolveSerial(avdName:)`.

**DSL target bundle ID**: the test panel has an iOS-only `iOS App:` field (hidden when no iOS simulator is active). Android's `uiautomator dump` ignores it. For iOS, blank = SpringBoard; populated = passed to `XCUIApplication(bundleIdentifier:)`. Auto-prefilled from `IOSProjectConfiguration.bundleIdentifier`, which `BuildPlayRunner.discoverIOSProject` extracts from `xcodebuild -showBuildSettings | grep PRODUCT_BUNDLE_IDENTIFIER` when the project is picked.

### Bottom panel UI

`ContentView` has a unified bottom panel (`BottomPanelView`) driven by `KittyFarmStore.bottomPanel: BottomPanel` (`.hidden`, `.logs`, `.tests`). The panel has a tab header with badges (error/warning counts for logs, pass/fail counts for tests) and hosts either `BuildLogPanelView` or `TestScriptView`. A build auto-switches to `.logs` only if the panel was hidden ‚Äî it won't interrupt an open test view.

`TestScriptView` uses a custom `CodeEditor` `NSViewRepresentable` (not `TextEditor`) because NSTextView's default quote/dash/spelling substitutions corrupt DSL scripts. All substitutions are explicitly disabled there.
